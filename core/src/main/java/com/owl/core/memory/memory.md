#### 完整提示词内容
```text

你现在需要严格遵循「刚性容量红线+优先级临时遗忘」规则，对现有的三层索引库内容，完成本次调用的记忆筛选与临时遗忘，确保带入的记忆内容不超过大模型上下文窗口总容量的30%，从根源避免爆窗口、上下文腐败。

3.  对话中用户仅引用、未做个性化修改/补充/专属决策的通用知识内容
4.  模型自身生成的、无用户专属信息的通用回答内容
5.  任何无需用户专属信息即可复现的公开知识内容

3.  对话历史中用户的核心诉求、历史决策、已确认的专属结论、用户与模型达成的个性化共识

1.  第一步：先确认容量红线
    先明确当前大模型的上下文窗口总token容量，计算出30%的记忆内容最大token上限，本次带入的所有记忆内容，绝对不能超过这个上限。

    ✅ 必须带入（P0优先级，永不临时遗忘）：
    1.  用户当前提问的核心检索锚点、本次对话的顶层目标导向索引库的全部内容
    2.  长期核心记忆库中的全部内容（用户长期偏好、禁忌、固定规则、核心目标）

    ❌ 绝对不带入（P3优先级，100%临时遗忘）：
    1.  所有B级低权重索引及对应归档内容
    2.  和当前任务核心目标完全无关的所有历史内容
    3.  超过7天未被激活的非核心索引
    4.  重复、冗余的历史内容
    5.  【新增】所有符合上文「通用知识过滤刚性规则」的大模型已知通用知识内容
    以上内容一律不放进本次上下文窗口，不占用任何token，仅留存于长期库（通用知识直接不归档），不做删除。

3.  第三步：容量超限处理
    如果按优先级带入后，总token量超过30%的容量红线，严格按以下顺序执行临时遗忘，直到符合容量要求：
    ①  先全部临时遗忘P2优先级的所有内容
    ②  再临时遗忘P1优先级中，时间最远、激活次数最少的内容，一条一条依次排除
    ③  绝对不能临时遗忘P0优先级的内容，仅可做极简语义压缩，不可删除核心信息。

4.  第四步：最终校验
    必须确保最终带入的记忆内容，同时满足以下条件：
    ✅ 总token量不超过窗口总容量的30%
    ✅ 完整覆盖P0优先级的所有核心内容
    ✅ 无任何P3优先级的无关内容
    ✅ 所有带入内容都和当前用户的提问/核心目标强相关
    ✅ 【新增】无任何大模型已知通用知识内容，所有带入内容均为用户专属个性化白名单内内容

【输出规范】
你必须依次输出：【本次容量红线】、【通用知识过滤结果】、【最终带入的记忆内容】、【本次临时遗忘的内容清单】，不输出任何额外的解释、说明、寒暄话术。
```

1.  **【本次容量红线】**：明确本次LLM窗口总容量、30%记忆内容的token上限；
2.  **【通用知识过滤结果】**：标注本次拦截的通用知识内容、过滤数量、拦截原因；
